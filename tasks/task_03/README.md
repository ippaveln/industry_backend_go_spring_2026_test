# fizzbuzz

![task 03](../../badges/tasks/task_03.svg)

В этом задании нужно реализовать классическую функцию FizzBuzz, но в виде функции, которую проверяют автотесты.

Цель: научиться читать тесты, реализовывать логику по спецификации и аккуратно работать с `error`.

---

Что нужно сделать

1) Откройте тест `Test_fizzBuzz` и посмотрите, какие случаи он проверяет.
2) Реализуйте функцию `fizzBuzz`, чтобы все тесты проходили.

---

Требования к функции

Функция должна иметь такую сигнатуру:

`func fizzBuzz(n int) (string, error)`

Правила:

- если `n` делится на 3 без остатка — вернуть `"Fizz"`
- если `n` делится на 5 без остатка — вернуть `"Buzz"`
- если `n` делится и на 3, и на 5 — вернуть `"FizzBuzz"`
- если `n` не делится ни на 3, ни на 5 — вернуть строковое представление числа (например, для `7` вернуть `"7"`)
- если `n` отрицательное — вернуть пустую строку `""` и ошибку (`error != nil`)

Важно:
- функция не должна печатать в stdout (никаких `fmt.Println`), она должна только возвращать значения
- тесты менять нельзя

---

Примеры ожидаемого поведения

- `fizzBuzz(9)`  -> `"Fizz"`, `nil`
- `fizzBuzz(10)` -> `"Buzz"`, `nil`
- `fizzBuzz(15)` -> `"FizzBuzz"`, `nil`
- `fizzBuzz(7)`  -> `"7"`, `nil`
- `fizzBuzz(-5)` -> `""`, `error`
