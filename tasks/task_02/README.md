Работа со строками (UTF-8)

![task 02](../../badges/tasks/task_02.svg)

Цель задания — научиться правильно работать со строками в Go, учитывая UTF-8, и понять разницу между байтами и рунами.

**Задача**

Реализуйте функцию `reverseRunes`, которая разворачивает строку **по рунам (Unicode code points)**, а не по байтам.

Сигнатура функции:

`func reverseRunes(s string) string`

Функция должна вернуть новую строку, в которой руны исходной строки идут в обратном порядке.

**Почему “по рунам”, а не “по байтам”**

В Go строка (`string`) — это последовательность байт в UTF-8. Если разворачивать строку побайтно, то для не-ASCII символов (например, кириллицы или иероглифов) получится некорректная UTF-8 последовательность.

Пример (идея):
- строку `"Hello"` можно развернуть и по байтам, и по рунам — результат будет корректным
- строку `"Привет"` нельзя разворачивать по байтам — UTF-8 “сломается”, потому что один символ занимает несколько байт

**Примеры**

- `reverseRunes("") -> ""`
- `reverseRunes("A") -> "A"`
- `reverseRunes("Hello, World!") -> "!dlroW ,olleH"`
- `reverseRunes("АБВГДЕ") -> "ЕДГВБА"`
- `reverseRunes("Hello, 世界") -> "界世 ,olleH"`

**Требования**

- Разворачивать именно **по рунам**
- Должно работать с ASCII и UTF-8 строками
- Пустая строка должна корректно обрабатываться
- Решение должно проходить тесты из репозитория (см. `go test`)
- Не изменяйте тесты (если явно не указано обратное в вашем курсе)

**Важно про “символы”**

В этом задании мы работаем с рунами (кодпоинтами), а не с “графемами” (визуальными символами, которые могут состоять из нескольких рун, например буква + диакритика). Это нормально: тесты проверяют именно разворот по рунам.
