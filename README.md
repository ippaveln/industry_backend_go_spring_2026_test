**Как выполнять задания**
1) Сделайте fork этого репозитория в свой GitHub-аккаунт.
2) Выполняйте задания в своём форке.

**Обновления репозитория**
В репозиторий могут прилетать обновления (фиксы, уточнения, изменения тестов и т.п.).
Если в исходном репозитории появились изменения — перед продолжением работы **обновите свой fork** и убедитесь, что решение проверяется на актуальной версии.

**Важно про изменения в репозитории**
Разрешены изменения **только** в файлах `solution.go` в папке каждого задания (например: `tasks/task_00/solution.go`).

Любые изменения **в любых других файлах/папках** считаются **невалидными** и **не будут засчитаны**, включая (но не ограничиваясь):
- правки тестов, `README.md`, конфигов, `go.mod/go.sum`, CI/CD, бейджей;
- добавление/удаление файлов;
- переименование файлов или изменение структуры директорий.

**Зависимости**
В решениях **запрещены любые внешние зависимости**.
Используйте **только стандартную библиотеку Go** (stdlib).
Любые импорты сторонних пакетов (включая обновления `go.mod/go.sum`) считаются **невалидными** и **не будут засчитаны**.

**Сдача**
После окончания выполнения задания (или набора заданий) отправьте Pull Request **из вашего форка в этот репозиторий**.

Требование к заголовку Pull Request:
- обязательно укажите `ИСУ xxxx`, где `xxxx` — ваш табельный номер (например: `ИСУ 123456`).
